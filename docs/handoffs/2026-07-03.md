# EFlux External User Participation Handoff - 2026-07-03

EFlux is an LLM-driven virtual-power-plant electricity-trading simulator with a FastAPI backend (`src/eflux`), React frontend (`frontend/`), and tests under `tests/`.
This handoff summarizes the external user participation work stream: what is currently built in the repo, and what remains unsettled.

- `git status --short` was clean before this handoff file was created.
- External participation spec: `docs/EXTERNAL_PARTICIPATION.md`; protocol docs: `docs/AGENT_SPEC.md` section 5 and 6.

## Done

### Spec and docs

- `docs/EXTERNAL_PARTICIPATION.md` defines the four-tier participation menu:
  - Tier 0: platform-hosted managed agents.
  - Tier A1: local agent exchanging order batches.
  - Tier A2: networked RL environment.
  - Tier A3: local LLM guidance.
- Cross-links exist from `docs/AGENT_SPEC.md` to the external participation doc.
- `docs/AGENT_SPEC.md` now documents Agent Protocol v1, the SDK/MCP surface, and Tier A3 external guidance.

### Tier 0 managed agents

- Owner-scoped managed VPP APIs are implemented in `src/eflux/api/routers/vpps.py`:
  - `GET /vpps/managed`
  - `GET /vpps/managed/{vpp_id}/performance`
  - `POST /vpps/managed`
  - `PATCH /vpps/managed/{managed_id}`
  - `DELETE /vpps/managed/{managed_id}`
- Live provisioning uses `provision_managed_vpp` in `src/eflux/simulator/scenarios.py`.
- Runtime scoping uses `Simulator.my_managed_vpps(owner_id)` in `src/eflux/simulator/runner.py`.
- Persistent definitions are stored on `vpps.is_managed` and `vpps.managed_config` in `src/eflux/db/models.py`; migration is `alembic/versions/0002_managed_vpp_columns.py`.
- Managed agents are rehydrated on backend startup by `_rehydrate_managed_vpps` in `src/eflux/api/main.py`.
- The managed-agent name reuse/orphan-row path is handled in `POST /vpps/managed`: uniqueness/quota is checked against live managed VPPs, and stale persisted rows with the same name are deleted before create.
- The My VPPs UI includes deployment/tuning/delete flows in `frontend/src/pages/MyVPPs.tsx`.

### Per-agent model selection and deploy params

- Curated model selection is implemented through `CURATED_MODELS` and `SharedLLM.client_for` in `src/eflux/agents/reflective/pool.py`.
- `GET /vpps/models` returns the curated model list in `src/eflux/api/routers/vpps.py`.
- Managed create/patch payloads carry `model`; the My VPPs page exposes a model dropdown.
- Managed deploy params include wind and `load_profile` through the shared VPP params schema.

### Chatroom and owner presence

- Public chat feed exists at `GET /market/chatter` in `src/eflux/api/routers/market.py`.
- LLM chat prompt construction and cleanup live in `src/eflux/agents/reflective/chat.py`.
- Runtime chat scheduling and generation live in `Simulator._maybe_chat`, `Simulator._generate_chat`, and `Simulator.post_chat` in `src/eflux/simulator/runner.py`.
- Conversational context is implemented: recent chat is passed into the prompt, agents can reply when mentioned, and reply streaks are capped so the room is not only replies.
- Owner chatroom presence is implemented:
  - `PUT /vpps/managed/{managed_id}/chat` sets chat style, color, and avatar.
  - `POST /vpps/managed/{managed_id}/say` posts as the owner through their managed agent.
  - Chat messages include `source`, `color`, and `avatar`.
  - `frontend/src/components/Chatroom.tsx` renders owner-typed messages (`source == "owner"`), avatars, and per-message color.
  - Integration coverage exists in `tests/integration/test_chatroom_api.py`.

### Lesson privacy

- Public `GET /market/reflections` deliberately omits `lesson` in `src/eflux/api/routers/market.py`.
- Owner-only managed performance still exposes the reflection timeline, including `lesson`, through `GET /vpps/managed/{id}/performance`.

### P1 / Tier A1 Agent Protocol v1

- Implemented in `src/eflux/api/routers/orders.py`:
  - `POST /orders/batch`
  - `GET /orders/open`
  - Existing single-order `POST /orders` and `POST /orders/cancel` remain.
- Batch envelope supports `protocol_version`, `idempotency_key`, `deadline`, `orders`, and `cancels`.
- Batch behavior is per-item/non-atomic, runs cancels first, enforces per-order ownership, and returns per-item accepted/rejected results.
- In-memory idempotency replay is implemented with an LRU-bounded cache.
- Per-account token-bucket rate limit is implemented for order batches: 120 burst, 2 tokens/sec refill, 429 when exceeded.
- Simulator support exists in `submit_external_batch` / `_submit_one_external`; book state read uses `MatchingEngine.open_orders_for_vpp`; tick id comes from `Simulator._last_tick_no`.

### Tier A3 external guidance

Status: built, not just an SDK stub.

- Owner-scoped guidance routes exist in `src/eflux/api/routers/vpps.py`:
  - `PUT /vpps/managed/{managed_id}/guidance`
  - `DELETE /vpps/managed/{managed_id}/guidance`
- Guidance payload accepts the `StrategyGuidance` shape plus optional `meta_control`, clamps/sanitizes server-side through `apply_external_guidance`, and returns the applied reflection entry.
- Guidance ingestion is rate limited with a token bucket of 10 burst and about 2/min sustained refill.
- Applying guidance swaps the managed agent to `ExternalStrategist` in `src/eflux/agents/reflective/strategist.py`; while active, the platform LLM strategist is not called.
- Guidance is persisted in `managed_config` (`guidance_mode`, `external_guidance`) and rehydrated on startup by `_rehydrate_managed_vpps`.
- `PATCH /vpps/managed/{id}` preserves external guidance across preference changes.
- My VPPs shows an externally steered badge when `guidance_source == "external"`.
- Integration coverage exists in `tests/integration/test_guidance_api.py`.

## Remaining / unsettled

- A2 networked RL environment is still not implemented:
  - Existing reusable pieces: `VPPPrimitiveEnv`, obs/action codec, and reward shape.
  - Missing: over-the-wire `reset`/`step` transport, sandbox training market, and step budgets.
- P5 leaderboard/evaluation remains partial:
  - Done: durable live-session snapshots, score v1, leaderboard/history APIs, Leaderboard UI, read-only benchmark artifact APIs.
  - Missing: hosted submit-for-evaluation service, held-out scenario queue, standardized evaluation harness wrapping the backtest runner.
- Per-participant audit segmentation remains open:
  - Event/audit streams are still effectively global, not partitioned into per-account provenance views for competition operations.
- Rate limits are partial:
  - Present for A1 order batches and A3 guidance.
  - Not yet comprehensive for magic-link/single-order paths, and no full model-cost metering.
- Collusion / wash-trade detection remains open.
- Open design decisions from `docs/EXTERNAL_PARTICIPATION.md` section 8:
  - Exact scoring formula and risk-adjustment terms.
  - One market or two: whether A2 training shares live infrastructure or always runs in a per-user sandbox. The spec recommends separate sandbox training.
  - Platform-vs-local LLM for A3: local guidance is built; whether users may consume platform LLM quota for A3 remains a product/cost decision.
  - Credit/cost model: free tiers, quotas, and billing treatment for model cost.
  - Durability policy: live leaderboard snapshots now persist, but the offline held-out backtest/evaluation story is still the recommended basis for serious ranking.

## How to run / operational notes

- Use the repo venv directly; do not assume `pip`:
  - `EFLUX_PV_PHYSICAL=false PYTHONPATH=src .env/bin/python ...`
  - Tests: `.env/bin/python -m pytest`
- Restart the no-reload backend after endpoint changes so router and startup wiring are actually picked up.
- Enable MCP with `uv sync --extra mcp`; alternatively use an ad-hoc `uv pip install` into `.env` if needed for local experimentation.
- Run MCP as `eflux-mcp` or `python -m eflux.mcp.server`; configure with `EFLUX_MCP_BASE_URL` and `EFLUX_MCP_API_KEY` or `EFLUX_MCP_EMAIL`.
- Managed agents are persistent definitions: they auto-restore on backend restart. Delete a managed agent to free/reuse its name.
- Live market state remains ephemeral by design; durable leaderboard data is captured separately through session/snapshot rows.
